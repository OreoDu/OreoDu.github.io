Part.2 How do the programs work inside the computer?

First, let’s have a quick look at the simplified computer structure.

(from Python for Everybody)

When we finish our code and store it in the secondary memory, the Python interpreter will parses the source code and interprets the instructions and the complier will translate the high-level source code into machine language and put the resulting machine language into a file for later execution.

Then when we execute the program, it will be stored in the main memory temporarily. The computer will retrieve the data form the input devices or the network, get the instructions form the main memory and do the computations in the center processing unit(CPU) with a high speed(billion times per second). In the end, the computer will send the results to the output devices.

Part. 4 Language structure

We put these language elements together to form different statements so that we can solve a certain problem. The way we organize the statements can be divided into different types of structures in order to make our program more effective. Generally, there are three types: sequential structure, conditional structure and iteration.

Python uses indentation to set the hierarchy of different compound statements instead of using{}. If the continuous statements maintains the same indentation then they belong to the same code block, which is equivalent to one specific function.

Ⅰ. Conditional structure
Sometimes, we have to separate our solutions into different branches and change the actions of the program according to some conditions. In python, we use if statement(use the keywords if, else, elif) to implement it.

· Chained conditionals: (two branches or more)

# calculate the payment

    print('Pease enter numeric input!')

elif: one of the conditions, else: catch the exceptions
(If more than one condition is true, only the first true branch executes.)

· Nested conditionals: 

if fh<0 :

Usually, we use chained conditionals rather than nested conditionals or simplify it with operators. “Flat is better than nested”. Because the nested structure will have a serious impact on the readability of code.

Ⅱ. Iteration
Repetitive tasks are the main work for computers. Python provides the while statement and for statement to let the computers do the tasks in a more efficient way.

· while statement (indefinite loops):

# output the max and min
smallest = None
largest = None
while True:
    num = input('Enter the num: ')
    if num == 'Done':
        break
    try:
        num = int(num)
    except:
        print('Invalid input!')
        continue
    if smallest is None or num<smallest:
        smallest = num
    if largest is None or num>largest:
        largest = num
print('The smallest:',smallest,'The largest:',largest)

break: If the condition is met, the program will jump out of the loop.

continue: Skip to the next iteration without finishing the body of the loop for the current iteration.

*Ⅲ. Functions and Modules *
In order to write high-quality code, we should eliminate the repetitive code which is doing the same type of operations. We can encapsulate those repetitive codes into a function module called “function” so that we “call” the “function” when we need it. Generally, function is a named and reusable sequence of statements that takes arguments as input, performs a computation and return results. (The naming rules of function are consistent with the naming rules of variables. )

# How to define a function? Example:calculate the factorial
def factorial(num):
    result =  1 
    for n in range (1,num+1):
        result *= n
    return result

Actually, factorial function is a built-in function in python, we can use such kind of functions directly without defining them. More built-in functions in python.

· Module
Maybe sometimes, we are faced with naming conflicts. When this happens, the latter definition will override the previous definition, which means that only one of the two functions with the same name is actually executed. At such a moment, it is better to define the function with the same name in different files. In Python, every file represents a module. We can use the import to import the specified module to distinguish different functions.

module1.py

def foo():
     print ('hello, world!' )

module2.py

def foo():
     print ('goodbye, world!')

test.py

However, besides the functions defined in the module we import, if there are some other executable codes, it’d be better to put those codes into the condition shown below so that the python interpreter will only execute them when the module is run directly and will not execute them when we import the module into other files.

def foo():
    print ('goodbye, world!')
# __name__ is an implicit variable in Python that represents the name of the module.
if  __name__ == ' __main__ ' :
    print ('hello, world!')
    foo()

· Syntax errors:

Ex.7: Write a program to find all the palindrome prime within 10000.

import math

def prime(arg):
    is_prime = True
    for i in range(2,int(math.sqrt(arg))+1):
        if arg % i == 0 :
            is_prime = False
            break
    return is_prime

def palindrome(arg):
    result = 0
    number = arg
    while arg > 0:
        result = arg % 10 + result*10
        arg //= 10
    return result == number

exist = False
for num in range(10, 10000):
    if prime(num) and palindrome(num):
        print('%d is a palindrome prime!' % (num))
        exist = True
if exist is False :
    print('There is no palindrome prime!')
